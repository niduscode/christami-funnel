Especificación técnica para construir un embudo digital que reemplace la dependencia de referidos y capte clientes calificados desde Meta Ads en 7 días.

01

Resumen ejecutivo

Qué construimos, qué problema resuelve y los números que esperamos mover.

El problema

Christami depende casi por completo de referidos para captar nuevos clientes. Esto limita el crecimiento, genera estacionalidad impredecible y deja la captación fuera del control del equipo. No hay un canal digital propio que opere de forma continua.

La solución

Un embudo digital de bajo costo y alta velocidad: anuncios en Meta dirigen tráfico a una landing con VSL (video sales letter), un formulario corto califica el interés, y un mensaje pre-llenado en WhatsApp lleva al lead al equipo de ventas para cierre por conversación o visita en terreno.

★ ALCANCE

Alcance y handoff

Este desarrollo entrega la infraestructura web del embudo. No incluye gestión de campañas, producción audiovisual ni acompañamiento publicitario. Léelo antes de cualquier otra cosa.

Incluido en este desarrollo

  • Landing page completa, responsive, con VSL embebido (placeholder hasta que el cliente provea el video final)
  • Formulario de calificación funcional, con validaciones y lógica de scoring
  • Base de datos Supabase con tablas, políticas RLS y panel admin básico para visualizar leads
  • Redirección automática a WhatsApp con mensaje prellenado y datos del lead
  • Meta Pixel instalado y configurado · eventos PageView, ViewContent, Lead, CompleteRegistration, Contact
  • Google Analytics 4 instalado con eventos personalizados
  • Notificación por email (Resend) cada vez que entra un lead nuevo
  • Dominio configurado, certificado SSL, deploy productivo en Vercel
  • Documentación de handoff para el equipo de marketing (ver sección 15)
  • Capacitación inicial de 1 hora al cliente o su equipo para que sepan dónde ver los leads

Fuera del alcance (otro proveedor)

  • Creación de cuentas en Meta Business, Instagram Business, Google Ads
  • Diseño y producción del VSL (video sales letter)
  • Diseño gráfico, copy o producción de anuncios
  • Configuración, carga de presupuesto o gestión de campañas Meta Ads / Google Ads
  • Optimización de pauta, segmentación de audiencias, A/B testing de creatividades
  • Manejo de objeciones por WhatsApp, scripts de cierre de venta
  • Estrategia de contenido en redes sociales
  • Soporte continuo de campañas o "acompañamiento" durante 90 días
  • Garantías sobre número de leads, tasa de conversión o ROAS
Por qué esta separación: mantenerla explícita protege a las tres partes — cliente, agencia de marketing y desarrollo. Cada uno conoce su responsabilidad y nadie asume tareas para las que no se contrató. El cliente debe contratar una agencia o freelancer de marketing para todo lo que está en la columna derecha.

Arquitectura del sistema

Flujo de datos desde el clic en el anuncio hasta la notificación al equipo de ventas.

Server Action en lugar de API route: menos boilerplate y validación con Zod en el mismo archivo. El INSERT a Supabase y el envío vía Resend ocurren en serie dentro del action; si Resend falla, el lead ya está guardado y se notifica por log.

create table leads (
  id uuid primary key default gen_random_uuid(),
  created_at timestamptz default now(),
  nombre text not null,
  email text,
  telefono text not null,
  ciudad text,
  tipo_proyecto text check (tipo_proyecto in ('cocina','dormitorio','living','oficina','otro')),
  presupuesto_rango text check (presupuesto_rango in ('menos_1m','1m_3m','3m_5m','mas_5m')),
  plazo text check (plazo in ('inmediato','1_3_meses','3_6_meses','solo_cotizando')),
  score int default 0,
  estado text default 'nuevo' check (estado in ('nuevo','contactado','calificado','ganado','perdido')),
  utm_source text,
  utm_campaign text,
  utm_medium text,
  notas text
);

-- Tabla interacciones
create table interacciones (
  id uuid primary key default gen_random_uuid(),
  lead_id uuid references leads(id) on delete cascade,
  tipo text check (tipo in ('whatsapp','email','llamada','visita')),
  fecha timestamptz default now(),
  contenido text
);

create index idx_leads_estado on leads(estado);
create index idx_leads_created on leads(created_at desc);
07

Políticas RLS

Row Level Security activo en ambas tablas. El formulario público inserta vía anon key; el admin lee y modifica con sesión.

Activación y políticas para leads

-- Activar RLS
alter table leads enable row level security;

-- INSERT público: cualquiera puede crear un lead desde el formulario
create policy "leads_insert_public"
  on leads for insert
  to anon, authenticated
  with check (true);

-- SELECT solo autenticados (admin del cliente)
create policy "leads_select_authenticated"
  on leads for select
  to authenticated
  using (true);

-- UPDATE solo autenticados
create policy "leads_update_authenticated"
  on leads for update
  to authenticated
  using (true)
  with check (true);

-- DELETE deshabilitado por defecto (no creamos policy)

Políticas para interacciones

alter table interacciones enable row level security;

-- Toda la tabla es de uso interno: sólo autenticados
create policy "interacciones_all_authenticated"
  on interacciones for all
  to authenticated
  using (true)
  with check (true);
Importante: el campo score se calcula en el server action antes del INSERT. Aunque la policy de INSERT acepta cualquier valor, el cliente nunca lo envía: viene siempre del servidor. Si más adelante queremos enforzarlo a nivel DB, agregar una before insert trigger.

Lógica de scoring

El score se calcula en el server action al insertar. Va de 0 a 100. Cada respuesta suma puntos según la cercanía a "lead caliente":

// app/lib/score.ts
export function calcScore(lead: LeadInput): number {
  let score = 0;

  // Plazo: cuanto antes, mejor
  const plazoPts = {
    inmediato: 40,
    '1_3_meses': 28,
    '3_6_meses': 12,
    solo_cotizando: 4,
  };
  score += plazoPts[lead.plazo] ?? 0;

  // Presupuesto: rangos altos pesan más
  const presupPts = {
    mas_5m: 35,
    '3m_5m': 25,
    '1m_3m': 15,
    menos_1m: 5,
  };
  score += presupPts[lead.presupuesto_rango] ?? 0;

  // Tipo de proyecto: cocina y dormitorio son los más rentables
  const tipoPts = { cocina: 15, dormitorio: 12, living: 10, oficina: 8, otro: 5 };
  score += tipoPts[lead.tipo_proyecto] ?? 0;

  // Bonus: dejó email (canal extra de contacto)
  if (lead.email) score += 5;

  // Bonus: viene de campaña pagada identificable
  if (lead.utm_source === 'meta') score += 5;

  return Math.min(score, 100);
}

Umbrales sugeridos

  • 80–100 Lead caliente · contactar en menos de 1 hora
  • 50–79 Lead tibio · contactar el mismo día
  • 0–49 Lead frío · email automático + seguimiento semanal

// app/lib/whatsapp.ts
const NUMERO = '56912345678'; // reemplazar con número real Christami

export function buildWaLink(lead: Lead): string {
  const msg = [
    `Hola, soy ${lead.nombre}.`,
    `Vi su anuncio y completé el formulario.`,
    `Proyecto: ${lead.tipo_proyecto}`,
    `Plazo: ${lead.plazo}`,
    `Presupuesto: ${lead.presupuesto_rango}`,
    `Ciudad: ${lead.ciudad}`,
  ].join('\n');

  return `https://wa.me/${NUMERO}?text=${encodeURIComponent(msg)}`;
}

Decisiones abiertas

Preguntas pendientes de resolver con el cliente o entre los dos antes de empezar.

Opciones evaluadas:

  • Cliente graba con celular — gratis, autenticidad alta, riesgo de calidad pobre.
  • Productora externa — calidad premium, costo $400–800 USD, plazo +1 semana.
  • Cliente graba dirigido por nosotros vía WhatsApp/Meet — balance ideal, costo $0, requiere 2 sesiones.

Pendiente: confirmar fecha de grabación y guion. Sugerencia: comenzar con un VSL placeholder + texto, lanzar campaña pequeña, reemplazar cuando esté listo el final.

Mercado objetivo es 100% Chile, por lo que .cl transmite cercanía y confianza local. Costo similar (~$22 USD/2 años). Si christami.cl está disponible, esa es la elección por defecto.

Pendiente: verificar disponibilidad en NIC y comprar.

El cuello de botella del embudo no es técnico, es humano. Si nadie responde rápido, el score 80+ se enfría en 30 minutos. Necesitamos definir:

  • Persona responsable de primera respuesta
  • Horario de cobertura (idealmente 9–21h lunes a sábado)
  • Plan B fuera de horario: mensaje automático con expectativa de respuesta

Pendiente: confirmar con el cliente.

Christami opera principalmente en la región X. La logística de visita en terreno limita el alcance. Recomendación: empezar segmentando Meta Ads sólo a la zona operativa para no quemar presupuesto en leads que no podemos atender.

Pendiente: confirmar radio operativo del cliente y configurar geo-targeting acorde.

Ley 19.628 de Chile exige consentimiento informado para tratamiento de datos personales. Acciones mínimas:

  • Checkbox de consentimiento en el formulario (no pre-marcado)
  • Página /privacidad con texto legal claro
  • Banner de cookies si usamos Pixel + GA4

Pendiente: redactar política de privacidad (template revisable) y decidir entre banner de cookies simple vs. herramienta tipo Cookiebot.

Si después de 14 días el CPL supera $12.000 CLP sostenidamente, opciones:

  • Iterar creativos (probar 3 nuevos cada semana)
  • Ajustar segmentación (intereses + lookalike de clientes actuales)
  • Cambiar el formulario a 4 campos en vez de 7 (más leads, menos calificación → revisar trade-off)
  • Cortar la campaña, volver a referidos y reconsiderar estrategia

Pendiente: acordar criterio de "stop loss" con el cliente antes del lanzamiento.

4 · Parámetros UTM esperados

La landing captura automáticamente utm_source, utm_campaign y utm_medium desde el query string y los guarda junto a cada lead en la base de datos. La agencia debe usar URLs con UTMs en todos sus anuncios:

https://muebleriachristami.cl/?utm_source=meta&utm_campaign=cocinas_q2&utm_medium=instagram

Convención sugerida:

  • utm_source — plataforma origen: meta, google, email, tiktok
  • utm_medium — formato/canal: instagram, facebook, reels, stories, search
  • utm_campaign — nombre interno de la campaña: cocinas_q2, dormitorios_invierno_2026, black_friday
Recomendación: mantener una hoja de cálculo con todas las URLs UTM generadas para evitar inconsistencias entre anuncios y poder cruzar datos en el panel admin.
